Identify Multi-Channel Sources: Consolidate unstructured and structured data
across all touchpoints, including CRM records, social media interactions, call center logs,
service requests, chat messages, and mobile money transaction APIs.
Assess Churn Types: Define specific target variables for the model based on
business logic:
Active vs. Passive Churn: Distinguish between explicit cancellations and non-renewal
of subscriptions without communication.
Voluntary vs. Involuntary Churn: Identify customers leaving due to dissatisfaction vs.
payment failures (e.g., insufficient funds/expired cards).
Revenue vs. Customer Churn: Separate the loss of total accounts from the loss of
specific revenue streams (e.g., plan downgrades).
Execute Data Cleaning Protocol:
Remove constant features (e.g., Count, Country) that exhibit zero variance.
Drop redundant identifiers and non-predictive labels like CustomerID and Phone
Number.
RETAIN geospatial features (Latitude, Longitude) and Churn Reason. Note: Retain
Churn Reason as a categorical feature despite high missingness (73%), as it is a
primary driver in SHAP analysis.
Convert Total Charges and similar variables from object/string types to numeric formats,
coercing errors to handle blank entries and feature sparsity.
Implement Imputation:
Apply median imputation for numerical features to mitigate the impact of right-skewed
spending distributions and billing anomalies.
Use "most frequent" (mode) imputation for categorical data to maintain the
distribution of service usage and demographics.

3. Class Imbalance and Leakage Prevention
‚óè‚Äã Strict Leakage Control:
‚óè‚Äã Split data into stratified training/validation/test sets (e.g., 80/20 or 64/16/20) before any
oversampling is performed.
‚óè‚Äã Ensure the test set reflects real-world imbalance (approx. 26.5% churn) to provide an
unbiased Brier Score and ROC-AUC.
‚óè‚Äã SMOTE Implementation: Address class imbalances in the training set using Synthetic
Minority Over-sampling.
‚óè‚Äã Set the neighborhood size to $k=5$ to balance local manifold structure with noise
reduction.
‚óè‚Äã Apply SMOTE exclusively to the training set after the stratified split.

Iteration 1 (Baseline): Implement and benchmark a "Model Zoo" including XGBoost,
Random Forest, LightGBM, Gradient Boosting, AdaBoost, Logistic Regression,
and Multi-layer Perceptron (MLP) .
MLP Architecture: Implement a 64-32-1 hidden layer structure using ReLU activation
and L2 regularization (Alpha tuning).
Optuna-based Optimization: Execute Bayesian optimization (50+ trials) to maximize
ROC-AUC .
Utilize a 3-fold stratified cross-validation on the training set.
Deploy a Median Pruner to handle trial efficiency and early stopping for
poor-performing configurations.
Iteration 2 (SHAP-Driven Pruning):
Generate SHAP values for the top-performing baseline to identify global feature
importance.
Retain only the top 20 features to prune the feature space.
Target Metric: Achieve a ~28% reduction in training time while maintaining or
improving validation AUC.
